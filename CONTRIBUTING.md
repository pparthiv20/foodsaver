# Contributing to Food Saver

Thank you for considering contributing to Food Saver! Here are some ways you can help:

## How to Contribute

### 1. **Report Bugs**
- Use the GitHub Issues tab
- Provide a clear description
- Include steps to reproduce and expected vs actual behavior

### 2. **Suggest Features**
- Open a GitHub Discussion or Issue
- Explain the feature and why it would be useful
- Include examples if applicable

### 3. **Submit Code**

#### Setup Development Environment
```bash
git clone https://github.com/your-username/food-saver-php.git
cd food-saver-php
```

#### Create a Branch
```bash
git checkout -b feature/your-feature-name
```

#### Make Changes
- Follow the existing code style
- Add comments for complex logic
- Test thoroughly

#### Submit Pull Request
1. Push to your fork
2. Open a Pull Request to `main`
3. Include a description of changes
4. Reference any related issues

## Code Standards

### PHP
- Use camelCase for functions and variables
- Use PascalCase for classes
- Always use prepared statements for database queries
- Add input validation and sanitization
- Include error handling

### JavaScript
- Use arrow functions where appropriate
- Add comments for complex logic
- Test in multiple browsers

### HTML/CSS
- Use semantic HTML5 elements
- Maintain responsive design
- Follow accessibility standards (WCAG 2.1)

## Testing

Before submitting a pull request:
1. Test all new features locally
2. Check for PHP errors
3. Verify database queries work correctly
4. Test on mobile devices
